Introduction

Linux is a powerful open-source operating system used in servers, desktops, and embedded systems.

It offers flexibility, security, and control over system resources.

Linux distributions vary, but most share core components like the kernel and shell.

Understanding Linux is essential for DevOps, system administration, and cloud computing.

Linux supports multitasking and multi-user environments.

uname -a man ls whoami pwd echo "Hello Linux"
  • Open-source
  • Secure
  • Customizable
  • Stable
  • Community-driven
File System

Linux uses a hierarchical file system starting from the root `/` directory.

Common directories include `/home`, `/etc`, `/var`, and `/usr`.

Everything in Linux is treated as a file, including devices.

Use `ls`, `cd`, and `mkdir` to navigate and manage files.

Hidden files start with a dot (`.`).

ls -la cd /etc mkdir new_folder touch file.txt rm file.txt

Permissions

Linux uses a permission model to control access to files and directories.

Permissions include read (r), write (w), and execute (x).

Use `chmod`, `chown`, and `ls -l` to manage permissions.

Each file has an owner and group.

Permissions are represented in symbolic and numeric formats.

chmod 755 script.sh chown user:group file.txt ls -l umask stat file.txt

Package Management

Linux distributions use package managers to install and update software.

Debian-based systems use `apt`, while Red Hat uses `yum` or `dnf`.

Packages are stored in repositories.

Use `apt update` and `apt upgrade` to keep your system current.

Package managers resolve dependencies automatically.

sudo apt install nginx apt-cache search apache yum list installed dnf update snap install code
  • apt
  • yum
  • dnf
  • snap
  • flatpak
Networking

Linux provides powerful tools for networking and troubleshooting.

Use `ip`, `ping`, and `netstat` to inspect network configurations.

Services are managed via `systemctl` and configuration files.

Firewall rules can be set using `iptables` or `ufw`.

SSH is commonly used for remote access.

ip a ping google.com netstat -tuln ufw enable ssh user@host
